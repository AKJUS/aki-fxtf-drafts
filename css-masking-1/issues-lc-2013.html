Comments on draft of 29 October 2013

Last call document: http://www.w3.org/TR/2013/WD-css-masking-1-20131029/.

This document is the disposition of comments on the CSS Masking Module Level 1 CR sent to the www-style, public-fx and www-svg mailing lists. The CSS Masking Module Level 1 CR was published DD Month 2013.

The following color coding convention is used for comments:

  • The CSS WG agreed with the reviewer.
  • The CSS WG disagreed with the reviewer.
  • The CSS WG disagreed with the reviewer and the reviewer raised a Formal Objection.

(There were no formal objections.)

#CommentAnswer
Move security model to CSP - Comment from Dirk Schulze on Thu, 4 Apr 2013; solution: fixed in the spec.
1

"CSS Masking and Filter Effects describe a security model for loading filter/masking/clipping resources from different origins. Speaking with some browser vendors it looks like these kind of security issues should be handled by the Content Security Policy (CSP) spec."

A security model belongs to the specification. This security model was added and is part of CSS Masking Module Level 1. However, the SVG WG decided to work on a module called SVG Integration which will define the fetching policies for SVG documents. Especially SVG documents loaded as source of further resources or as image.

Layered Masking - Comment from Dirk Schulze on Sat, 31 Sep 2013.
2

The prefixed implementation of CSS WebKit in Safari support multiple mask layers - similar to multiple background layers for the 'background-image' property. CSS Masking should support layers as well.

It was unclear how authors expect mask layers to interact with each other (union, intersection, ...). Another proposal is to composite different mask layers with each other and let the authors decide which compositing mode to use. Resolution from the CSS WG: defer layered masking to a future version of the spec.

Reference elements by selectors - Comment from Brian Birtles on Mon, 23 Jul 2012.
3

Allow selecting mask and clipPath elements by CSS selectors in addition to fragment identifiers. Proposed: element(<compound-selector>) function and child keyword.

The specifications CSS GCPM and CSS3 UI allow element selecting. The CSS WG is investigating to find a common solution that can be used by all specifications. Element selecting deferred to a future level of the specification by editor.

No effect of masking and clipping on focus ring - Comment from Dirk Schulze on Wed, 20 Nov 2013.
4

From the minutes: "
krit: I would like to discuss outline property.
krit: Right now the spec says that every painted thing gets filtered/
blended.
krit: Turns out that outline property is also used for focus rings,
and that is very important a11y. For this reason, it seems not
useful to have the focus ring be filtered etc.
krit: We should masking/filters/blending not affect the focus ring."

Resolution of CSS WG: Effects affect scrollbars and focus rings. We may work on controls later.
