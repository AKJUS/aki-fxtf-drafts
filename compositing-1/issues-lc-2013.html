Last call document: http://www.w3.org/TR/2013/WD-compositing-1-20131010/.

This document is the disposition of comments on the Compositing and Blending CR sent to the www-style, public-fx and www-svg mailing lists. The Compositing and Blending CR was published 10 October 2013.

The following color coding convention is used for comments:

  • The CSS WG agreed with the reviewer.
  • The CSS WG disagreed with the reviewer.
  • The CSS WG disagreed with the reviewer and the reviewer raised a Formal Objection.

(There were no formal objections.)

From what you describe this is exactly the behavior that we agreed on to the last two F2F meetings. The behavior that you describe as "parent layer" is described as isolation to the next stacking context. I would not say that this is confusing. It is something that you can experience with isolation in other graphic applications as well. In future levels we get a new property value for 'isolation' that allows authors to blend with the whole backdrop, but first browsers need to catch up.

Comment from Chris Lilley on Fri, 6 Dec 2013; answer on www-svg.
4

the test plan includes tests in SVG and HTML, but compositing for SVG and HTML is deferred to level 2?

The only 'compositing' part that the spec currently defines, is for Canvas 2D. Compositing was stripped from the CSS properties because it required major changes in the way browsers draw. It will be back for level 2 and either we will change the model so it can be implemented or we convince the browsers to redo their graphics engines.
